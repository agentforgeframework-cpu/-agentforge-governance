# Purpose

This quickstart explains how to use the AgentForge governance repository.

The governance repository contains foundational standards, principles, and governance guidance that apply across the AgentForge ecosystem.

---

# Basic Use

1. Start with `README.md`.
2. Review the applicable governance document in `/governance/`.
3. Treat governance documents as authoritative unless a later production version replaces them.
4. Use support materials in `/documents/` and `/media/` for explanation, communication, and adoption support.
5. Do not treat support materials as replacements for the governance documents themselves.

# Adoption Pattern

Projects may adopt governance standards by referencing the canonical file path.

Example:

> This project operates under HIC-001: Human-in-Command Standard.

Projects may copy governance standards locally when needed for AI project context, but the canonical source remains this repository.

---

# Guidance

Governance documents should remain short, stable, and authoritative.

Implementation details, examples, operational patterns, and interpretive guidance should be placed in supporting materials, not inside the standard unless they are truly part of the standard.

---

# Recommended Project Reference

When another project adopts HIC-001, include a brief reference such as:

> This project recognizes HIC-001: Human-in-Command Standard. Systems may assist operations. Humans retain operational authority, judgment, and accountability.
