# Purpose

This folder contains the machine-readable operational tools and capability components for the kit.

---

# What Belongs Here

This folder may contain:

- operational tools
- workflows
- procedures
- executable code
- templates intended for machine use
- structured machine-readable instructions
- implementation utilities

---

# What Does Not Belong Here

Human instructions, explanatory documents, printable guides, and user-facing support materials belong in `/documents/`.

Examples belong in `/examples/`.

Validation evidence belongs in `/validation/` when used.

---

# File Naming

Operational tools/components should normally use lowercase names.

Examples:

```text
cleanup.md
analyze.md
forecast-tool.md
```
